Format of library files

Format of index files

The general structure of index file is as follows (see examples):

<html>
<head>
<meta http-equiv="Content-Type" content="text/html; charset=windows-1251">
<link rel="stylesheet" type="text/css" href="URI">
</head>
<body>
<div>
<a href="URI">upward</a>,
<a href=".">fold</a>,
<a href="URI">quickaccess1</a>,
<a href="URI">quickaccess2</a>
</div>
<h1>title</h1>
Main body
</body>
</html>

Allowed quick access links:

  • sci – to corresponding project or Science entry
  • wiki – to Wikipedia-like page

The main body may consist of the following entries:

  • Subheading of levels h2 and h3
    <hx>title</hx>
  • List of links, which must follow strictly after a heading or a subheading (keywords may be repeated)
    <ul>
    <li><a href="URI">subtopic1</a>
    <li><a href="URI">subtopic2</a>
    <li><i>keyword</i> <a href="URI">item1</a>, <a href="URI">item2</a>
    <li><i>keyword</i> <a href="URI">item3</a>, <a href="URI">item4</a>
    </ul>
  • Bibitem entry (class "g" indicates depreciated entry)
    <p>bibitem</p>
  • Separator
    <hr>

The possible values of keyword in a list of links are as follows:

  • see also related topics (links to the corresponding index files)
  • journals (and news portals)
  • people
  • org (organizations etc.)

Filenaming conventions

Filename structure is <prefix><author><YY><suffix>.

Prefix indicates document type:

indexed articleno suffix
not indexed article_
reviewRev_
reference manualRef_
encyclopediaEnc_
book or monographBook_
lecture or introductionLec_
letter or short noteLet_
taskbookTask_
collection of articlesCol_
PhD thesisPhD_

Author is surname of first author or editor with omitted spaces.

Suffix provides additional information.

multiple articles in yeara,b,c,...
multiple volumesv1,v2,...
only part of documentp or p1,p2,...
draftdraft
erratumerr
corrigendumcor
commentcom
replyrep
